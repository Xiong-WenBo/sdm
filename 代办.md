### 角色说明

#### 1️⃣ 超级管理员 (SUPER_ADMIN)
- 拥有所有功能模块的完整权限
- 管理所有用户、楼栋、房间、学生
- 查看所有数据，进行系统级配置

#### 2️⃣ 宿管 (DORM_ADMIN)
- 管理**本楼栋**的房间、宿舍分配、查寝、报修
- 查看本楼栋学生请假信息（无审批权）
- **不可**管理用户、楼栋、学生信息（增删改查）
- 可在房间管理、查寝管理中关联查看学生数据

#### 3️⃣ 辅导员 (COUNSELOR)
- 管理**本班级**的学生信息
- 查看本班级学生查寝记录
- 审批本班级学生请假
- **不可**管理用户、楼栋、房间、报修

#### 4️⃣ 学生 (STUDENT)
- 查看个人数据
- 提交报修申请
- 提交请假申请

## 📝 开发规范要求

1. 每个功能点开发遵循「设计接口 → 实现后端 → 测试接口 → 开发前端 → 联调」流程
2. 后端权限控制使用 Spring Security 的@PreAuthorize 注解
3. 所有列表页需实现分页、筛选功能，核心数据支持导出
4. 密码类敏感信息需做加密处理，前端表单需做合法性校验
5. 每次提交前需向用户确认具体内容
6. 不自动通过 shell 命令运行前后端（用户手动在 IDEA 和 VS Code 中启动）
